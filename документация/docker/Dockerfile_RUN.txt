RUN [OPTIONS] <command> ...   - shell форма
RUN [OPTIONS] [ "<command>", ... ] - exec форма

Инструкция RUN выполнит любые команды для создания нового слоя поверх текущего образа.
Добавленный слой используется на следующем шаге в Dockerfile.

Инструкций RUN может быть много, каждая добавляет новые изменения и фиксирует слой (сохраняет)

OPTIONS:
--mount
--network
--security

#1  Кэш для RUN инструкций может быть признан недействительным с помощью --no-cache флага, например docker build --no-cache.

=========================================================================

    OPTIONS

    1. RUN -mount
    RUN --mount=[type=TYPE][,option=<value>[,option=<value>]...]

    TYPE:
        1. bind(по умолчанию) Каталоги контекста привязки-монтирования (только для чтения).
        2. cache Смонтируйте временный каталог для кэширования каталогов для компиляторов и менеджеров пакетов.
        3. tmpfs	Смонтируйте a tmpfs в контейнере сборки.
        4. secret	Разрешить контейнеру сборки получать доступ к защищенным файлам, таким как закрытые ключи,
        без их включения в образ или кэш сборки.
        5. ssh	Разрешить контейнеру сборки доступ к SSH-ключам через SSH-агенты с поддержкой парольных фраз.

    RUN --mount позволяет создавать монтирования файловой системы, к которым сборка может получить доступ.
    Это можно использовать для:

    Создать привязку монтирования к файловой системе хоста или другим этапам сборки
    Доступ к секретам сборки или сокетам ssh-agent
    Используйте постоянный кэш управления пакетами для ускорения сборки

    2. RUN --network
    RUN --network=TYPE позволяет контролировать, в какой сетевой среде выполняется команда.

    TYPE:
    1. default(по умолчанию) Запуск в сети по умолчанию.
    2. none	Запускать без доступа к сети.
    3. host	Запуск в сетевой среде хоста.

    3. RUN --security=<sandbox|insecure>
